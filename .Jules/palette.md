## 2024-05-18 - Added ARIA labels to icon-only buttons
**Learning:** Found that multiple components (WaitlistModal, Resources, NewsletterAdminInner) use Lucide React icons for interactive buttons (like copy to clipboard or showing/hiding secrets/passwords) without providing an accessible name for screen readers. The `lucide-react` icons inside raw `<button>` elements or custom UI `<Button>` elements do not expose any text content.
**Action:** When adding icon-only buttons, always supply an `aria-label` attribute on the button element itself that clearly describes its action. If the action toggles state (e.g. eye icon for showing/hiding a password, or a copy icon that turns into a checkmark), the `aria-label` should update dynamically to reflect the current state (e.g. `aria-label={copied ? 'Copied referral link' : 'Copy referral link'}`).
